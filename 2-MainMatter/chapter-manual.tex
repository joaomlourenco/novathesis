%%=======================================================================
\section{Introduction}
\label{sec:intro_manual}
%%=======================================================================

This chapter is a complete guide to the \novathesis{} template.  It covers
everything from the first-time setup to the fine-tuning of covers, fonts,
citation styles, and the document workflow.  All user customization flows
through the files inside the \verb!0-Config/! folder — the rest of the
project tree is never edited directly.

For a reference to the automated build assistant \verb!.Build/build.py!,
see Chapter~\ref{app:build_py}.  For a guide to the cover page system,
see Chapter~\ref{app:covers}.

%%-----------------------------------------------------------------------
\subsection{Local \hologo{LaTeX} Installation}
\label{sub:local_install}
%%-----------------------------------------------------------------------

\novathesis{} requires a \emph{full} \hologo{LaTeX} installation.  The two
major distributions are:

\begin{description}
  \item[\TeXLive] The reference distribution maintained by \hologo{LaTeX} developers.
    The full scheme occupies around 5\,GB but includes every package you will
    ever need.  Strongly recommended unless disk space is very tight.
    Download from \url{https://www.tug.org/texlive/}.

  \item[\MikTeX] Installs a minimal base and downloads missing packages on the
    fly.  A good choice when disk space is limited.  Packages installed on
    the fly may occasionally fail, so keep the installation up to date.
    Download from \url{https://miktex.org/}.
\end{description}

After installation, update all packages via the \emph{\TeXLive Manager} or
the \emph{\MikTeX Console} before compiling the template for the first time.

\paragraph{Recommended engine.}  \hologo{LuaLaTeX} is the recommended engine for most
schools because it handles Unicode fonts natively.  \hologo{XeLaTeX} is a close
alternative.  \hologo{pdfLaTeX} works but cannot load OpenType/TrueType fonts.

\paragraph{School-specific fonts.}  Some Schools or font-styles require fonts not available in \hologo{LaTeX}.  In normal conditions, they will be downloaded automatically.  However, if they are not, you may download the necessary fonts from \url{https://github.com/joaomlourenco/novathesis-extras/tree/main/Fonts}, unzip (if necessary), and copy the \verb!*.ttf! files to \verb!NOVAthesisFiles/FontStyles/Fonts/!.


%%-----------------------------------------------------------------------
\subsection{Downloading the Template}
\label{sub:download}
%%-----------------------------------------------------------------------

Clone the GitHub repository (recommended — you get all update history):
\begin{bashcode}
git clone --depth=1 https://github.com/joaomlourenco/novathesis.git
\end{bashcode}

Or download the latest ZIP directly:
\begin{bashcode}
# Click "Code > Download ZIP" on https://github.com/joaomlourenco/novathesis
\end{bashcode}

The top-level folder is your working directory.  Rename it to anything
you like.  Open \verb!template.tex! in your editor and compile it — you
should get a \gls{PDF} immediately with the default settings (\gls{FCT}
PhD in English).

%%-----------------------------------------------------------------------
\subsection{Overleaf}
\label{sub:overleaf}
%%-----------------------------------------------------------------------

\Overleaf{} users can import the template directly from the GitHub link.
Once imported:

\begin{enumerate}
  \item Set the \emph{Compiler} to \textbf{\hologo{LuaLaTeX}} (Menu → Compiler).
  \item Set the \emph{Main document} to \verb!template.tex!.
  \item Set the \emph{Bibliography tool} to \textbf{Biber}
    (Menu → Bibliography).
  \item Press \emph{Recompile}.
\end{enumerate}

On \Overleaf{} you cannot run \texttt{make} or \texttt{build.py};
all customization must be done by editing the \verb!0-Config/! files
directly (see Section~\ref{sec:project_structure}).

\section{Configuring the Document (\texttt{1\_novathesis.tex})}

%%-----------------------------------------------------------------------
\subsection{Essential Options}
\label{sub:essential_options}
%%-----------------------------------------------------------------------

\begin{description}
  \item[\texttt{doctype}] The type of document being written.
    Default: \texttt{phd}.
\begin{latexcode}
\ntsetup{doctype=phd}      % PhD thesis
\ntsetup{doctype=phdplan}  % PhD thesis plan
\ntsetup{doctype=phdprop}  % PhD thesis proposal
\ntsetup{doctype=msc}      % MSc dissertation
\ntsetup{doctype=mscplan}  % MSc dissertation plan
\ntsetup{doctype=bsc}      % BSc report
\ntsetup{doctype=plain}    % Other report
\end{latexcode}

    The \texttt{*plan} and \texttt{*prop} document types produce a
    \emph{simplified} front matter: only the abstracts and the AI disclosure
    statement are included.  Pages such as the dedication, acknowledgements,
    quote, and evaluation committee are suppressed regardless of the
    \texttt{docstatus} setting.

  \item[\texttt{school}] The school identifier.  This selects the
    institutionally-compliant cover layout.  Default: \texttt{nova/fct}.
\begin{latexcode}
\ntsetup{school=nova/fct}      % NOVA FCT (default)
\ntsetup{school=nova/fcsh}     % NOVA FCSH
\ntsetup{school=nova/itqb/green}
\ntsetup{school=ulisboa/ist}
\ntsetup{school=uminho/eeng}
% ... see 1_novathesis.tex for the full list
\end{latexcode}

  \item[\texttt{docstatus}] The workflow stage.  Controls which front-matter
    pages are included.  Default: \texttt{working}.  See
    Section~\ref{sec:status_workflow} for the full implications of each value.
\begin{latexcode}
\ntsetup{docstatus=working}      % drafting (default)
\ntsetup{docstatus=provisional}  % submitted; no committee yet
\ntsetup{docstatus=final}        % post-examination; committee, book spine, PDF/A-2b
\end{latexcode}

  \item[\texttt{lang}] Main language of the document body (ISO 3166-1 alpha-2
    code).  Default: \texttt{en}.
\begin{latexcode}
\ntsetup{lang=en}   % English (default)
\ntsetup{lang=pt}   % Portuguese
\ntsetup{lang=de}   % German
% cat, cz, dk, es, fr, gr, it, nl, pl, sk, uk also available
\end{latexcode}

  \item[\texttt{media}] Target output medium.  Default: \texttt{screen}.
\begin{latexcode}
\ntsetup{media=screen}  % equal margins, colored links (default)
\ntsetup{media=paper}   % binding margins, black links
\end{latexcode}
\end{description}

%%-----------------------------------------------------------------------
\subsection{Appearance Options}
\label{sub:appearance_options}
%%-----------------------------------------------------------------------

\begin{description}
  \item[\texttt{style/font}] Body font family.  Default: \texttt{newpx}
    (Palatino-like).  Options marked with~(*) require \hologo{LuaLaTeX} or \hologo{XeLaTeX}.
\begin{latexcode}
\ntsetup{style/font=newpx}        % Palatino-like (default)
\ntsetup{style/font=libertine}    % Linux Libertine
\ntsetup{style/font=opensans}     %
\ntsetup{style/font=arial}        % (*) Arial
\ntsetup{style/font=calibri}      % (*) Calibri
\ntsetup{style/font=newsgott}     % (*) NewsGotT (UMINHO)
% Any standard LaTeX font package name is also accepted.
\end{latexcode}

    For system fonts (\hologo{LuaLaTeX}/\hologo{XeLaTeX} only), provide the font name and
    attributes explicitly:
\begin{latexcode}
\ntsetup{style/font=Times New Roman}
\ntsetup{style/font/attributes={
  Extension       = .ttf,
  Ligatures       = TeX,
  UprightFont     = *,
  ItalicFont      = * Italic,
  BoldFont        = * Bold,
  BoldItalicFont  = * Bold Italic,
}}
\end{latexcode}

  \item[\texttt{style/chapter}] Chapter heading style.  Default: \texttt{bar}.
\begin{latexcode}
\ntsetup{style/chapter=bar}          % (default)
\ntsetup{style/chapter=bar-compact}  % less vertical space
\ntsetup{style/chapter=elegant}      % classical
\ntsetup{style/chapter=bluebox}
\ntsetup{style/chapter=compact}      % minimal vertical space
\ntsetup{style/chapter=ist}          % ULISBOA-IST
\ntsetup{style/chapter=fmv}          % ULISBOA-FMV
\end{latexcode}

  \item[\texttt{color/links}] Color of hyperlinks in \texttt{screen} media.
    Default: a dark blue.  Any color name from the
    \texttt{xcolor} package works.
\begin{latexcode}
\ntsetup{color/links=SteelBlue}
\ntsetup{color/links=black}   % effectively disables colored links
\end{latexcode}

  \item[\texttt{style/url}] URL font style.
\begin{latexcode}
\ntsetup{style/url=same}  % use main body font for URLs
\end{latexcode}

  \item[\texttt{numbering/floats}] Float (figure/table) numbering scheme.
    Default: \texttt{chapter} (\emph{chapter.number}).
\begin{latexcode}
\ntsetup{numbering/floats=chapter}     % 1.1, 1.2, 2.1 … (default)
\ntsetup{numbering/floats=sequential}  % 1, 2, 3 …
\end{latexcode}
\end{description}

%%-----------------------------------------------------------------------
\subsection{Print Control Options}
\label{sub:print_options}
%%-----------------------------------------------------------------------

These options control which supplementary pages are included in the
generated PDF.  Many have defaults that vary with \texttt{school} and \texttt{docstatus}:

\begin{description}
  \item[\texttt{print/frontpage}] Print a front page (second cover).
    Default: \texttt{false}.
\begin{latexcode}
\ntsetup{print/frontpage=true}
\end{latexcode}

  \item[\texttt{print/committee}] Print the evaluation committee.
    Default: \texttt{false} for \texttt{working}/\texttt{provisional};
    \texttt{true} for \texttt{final}.
\begin{latexcode}
\ntsetup{print/committee=true}   % force on
\ntsetup{print/committee=false}  % force off
\end{latexcode}

  \item[\texttt{print/statement}] Print the declaration/integrity statement.
    Default: \texttt{false}.
\begin{latexcode}
\ntsetup{print/statement=true}
\end{latexcode}

  \item[\texttt{print/copyright}] Print the copyright notice.
    Default: \texttt{true}.
\begin{latexcode}
\ntsetup{print/copyright=false}
\end{latexcode}

  \item[\texttt{spine/layout}] Print the book spine.
    Default: \texttt{false} for \texttt{working} and \texttt{provisional};
    \texttt{trim} (printed) for \texttt{final}.
\begin{latexcode}
\ntsetup{spine/layout=false}  % suppress spine even in final
\ntsetup{spine/layout=trim}   % force spine even in provisional
\end{latexcode}

  \item[\texttt{print/index}] Print a word index at the end.
    Default: \texttt{false}.
\begin{latexcode}
\ntsetup{print/index=true}
\end{latexcode}

  \item[\texttt{print/timestamp}] Print a generation timestamp on the cover.
    Default: \texttt{true} for \texttt{working}; \texttt{false} otherwise.
\begin{latexcode}
\ntsetup{print/timestamp=false}
\end{latexcode}

  \item[\texttt{print/webography}] Title for a separate bibliography section
    that collects only \texttt{@online} entries.  When empty (default), all
    references are printed together.
\begin{latexcode}
\ntsetup{print/webography=Webography}
\end{latexcode}

  \item[\texttt{print/aidisclosure}] Print an AI use disclosure statement.
    Default: \texttt{aidisclose} (uses \verb!1-FrontMatter/7-aidisclose.tex!
    with the \texttt{aidisclose} package).
\begin{latexcode}
\ntsetup{print/aidisclosure=false}            % suppress
\ntsetup{print/aidisclosure=aidisclose}       % default package
\ntsetup{print/aidisclosure=aidisclosure.tex} % custom file
\end{latexcode}
\end{description}

%%-----------------------------------------------------------------------
\subsection{Language Options}
\label{sub:language_options}
%%-----------------------------------------------------------------------

\begin{description}
  \item[\texttt{abstractorder}] Ordered list of languages for the abstract
    sequence.  Default: \texttt{\{lang, en\}} when \texttt{lang}$\neq$\texttt{en};
    otherwise \texttt{\{en, pt\}}.
\begin{latexcode}
\ntsetup{abstractorder={en,pt}}
\ntsetup{abstractorder={en,pt,uk,gr}}   % add Ukrainian & Greek
\end{latexcode}

  \item[\texttt{lang/cover}] Language for the cover and front page.
    Default: same as \texttt{lang}.
\begin{latexcode}
\ntsetup{lang/cover=pt}   % Portuguese cover even if body is English
\end{latexcode}

  \item[\texttt{lang/copyright}] Language for the copyright message.
    Default: same as \texttt{lang}.

  \item[\texttt{lang/extra}] Additional languages loaded in the document
    (besides the main language and abstract languages).
\begin{latexcode}
\ntsetup{lang/extra={de,es}}
\end{latexcode}
\end{description}

%%-----------------------------------------------------------------------
\subsection{Sustainable Development Goals (SDGs)}
\label{sub:sdgs}
%%-----------------------------------------------------------------------

NOVAthesis can print the United Nation's SDG icons at the bottom of the abstract page(s).  All three options
below should be set together:
\begin{latexcode}
\ntsetup{print/sdgs/list={3,7,13}}    % SDG numbers to print
\ntsetup{print/sdgs/type=inverted}    % inverted | normal | mono
\ntsetup{print/sdgs/size=2cm}         % icon size
\ntsetup{print/sdgs/hspace=5mm}       % horizontal gap between icons
\ntsetup{print/sdgs/vspace=4mm}       % vertical gap below icons
\end{latexcode}

The full list of SDG numbers and their titles is reproduced in
\verb!0-Config/1_novathesis.tex! as comments for quick reference.

\section{Bibliography (\texttt{2\_biblatex.tex})}
\label{sec:bibliography}
%%=======================================================================

\novathesis{} uses \texttt{biblatex} with the \texttt{biber} backend.

%%-----------------------------------------------------------------------
\subsection{General Behavioral Options}
\label{sub:bib_general}
%%-----------------------------------------------------------------------

\begin{latexcode}
\ntbibsetup{
  backend      = biber,        % recommended; 'bibtex' also works
  giveninits   = true,         % print initials only
  maxbibnames  = 1000,         % list all authors (no 'et al')
  backref      = true,         % show page numbers for each citation
  backrefstyle = three,        % compact back-reference style
  urldate      = iso,          % ISO dates for URL access
  date         = iso,          % ISO dates for publications
}
\end{latexcode}

%%-----------------------------------------------------------------------
\subsection{Choosing a Citation Style}
\label{sub:bib_style}
%%-----------------------------------------------------------------------

Uncomment exactly one style block in \verb!0-Config/2_biblatex.tex!:

\begin{description}
  \item[Numeric (sorted by first author)] The default.  Citations like [1]:
\begin{latexcode}
\ntbibsetup{style=numeric-comp, sortcites=true, sorting=nyt}
\end{latexcode}

  \item[Numeric (citation order)] Sequential citations [1], [2], \ldots:
\begin{latexcode}
\ntbibsetup{style=numeric-comp, sortcites=true, sorting=none}
\end{latexcode}

  \item[Alpha] Citations like [Lou24]:
\begin{latexcode}
\ntbibsetup{style=alphabetic, sortcites=true, sorting=nyt}
\end{latexcode}

  \item[Author-year] Citations like (Lourenço, 2024):
\begin{latexcode}
\ntbibsetup{style=authoryear-comp, sortcites=true, sorting=nyt}
\end{latexcode}

  \item[Discipline-specific] IEEE, APA, Vancouver, Chicago are also
    available — see the style blocks in \verb!2_biblatex.tex!.
\end{description}

%%-----------------------------------------------------------------------
\subsection{Webography (Separate Online Bibliography)}
\label{sub:webography}
%%-----------------------------------------------------------------------

To collect all \verb!@online! entries in a separate section, set a title:
\begin{latexcode}
\ntsetup{print/webography=Webography}
\end{latexcode}
When this option is active, \verb!@online! references are excluded from the
main bibliography list and printed in their own section at the same location.

\section{Document Status Workflow}
\label{sec:status_workflow}
%%=======================================================================

The \texttt{docstatus} option controls which pages are included and
how the cover is rendered.  Table~\ref{tab:docstatus} summarises
the differences.

\begin{table}[htb]
  \caption{Effect of \texttt{docstatus} on the generated document.}
  \label{tab:docstatus}
  \centering\small
  \begin{ShadedTabular}{lccc}
    \toprule
    \textbf{Feature} & \texttt{working} & \texttt{provisional} & \texttt{final} \\
    % \midrule
    Timestamp on cover      & yes & no  & no  \\
    Dedication / quote      & no  & yes & yes \\
    Acknowledgements        & no  & yes & yes \\
    Integrity statement     & no  & yes & yes \\
    Evaluation committee    & no  & no  & yes \\
    Book spine              & no  & no  & yes \\
    PDF/A-2b output         & no  & no  & yes \\
    \bottomrule
  \end{ShadedTabular}
\end{table}
\noindent See Section~\ref{sub:pdfa} for details on the PDF/A archival output.

A typical workflow:
\begin{enumerate}
  \item Write with \texttt{docstatus=working} (fast, no distractions).
  \item Switch to \gls{msc} when submitting to the examination
    jury — a clean \gls{PDF} without the committee members.

  \item Switch to \texttt{final} after the examination — includes the
    committee, the book spine, and produces a PDF/A-2b archival document
    (see Section~\ref{sub:pdfa}).
\end{enumerate}

%%-----------------------------------------------------------------------
\subsection{PDF/A Archival Output}
\label{sub:pdfa}
%%-----------------------------------------------------------------------

When \verb!\ntsetup{docstatus=final}! is set, \novathesis{} automatically
produces a \textbf{\gls{PDF}/A-2b} conformant document — the archival-quality
format required by most institutional repositories and digital libraries.
No configuration beyond setting \texttt{docstatus=final} is needed.

\medskip
\noindent\textbf{How it works (single-pass).}
\novathesis{} loads the \texttt{pdfx} package during the normal \LaTeX{}
preamble, just before \texttt{hyperref}.  This single-pass approach requires
no extra compilation run beyond the usual cross-reference and bibliography
passes performed by every \LaTeX{} document.

The XMP metadata packet (title, author, language) is written from the
cover-page data defined in \verb!0-Config/3_cover.tex! and embedded into
the PDF automatically.

\medskip
\noindent\textbf{PDF/A standard choice.}
\novathesis{} targets \textbf{PDF/A-2b} (not~1b).  PDF/A-1b prohibits
transparency, which is incompatible with the TikZ cover-page graphics used
by virtually all supported schools.  PDF/A-2b lifts this restriction while
remaining accepted by all major institutional repositories.

\medskip
\noindent\textbf{XMP metadata.}
The thesis title, author name, and language are taken directly from the
cover-page data in \verb!0-Config/3_cover.tex! and passed to \texttt{pdfx},
which embeds them in the required XMP packet.  No additional configuration
is needed.

\medskip
\noindent\textbf{Images and colour spaces.}
The most common cause of PDF/A validation failures is an embedded image
with a device-dependent colour space (plain RGB or CMYK) and no ICC
profile.  PDF/A-2b requires every included image to carry an embedded
ICC profile or use a calibrated colour space.  Practical guidance by
format:

\begin{description}
  \item[JPEG / JPG] Most cameras and photo-editors embed an sRGB ICC
    profile automatically.  Screenshots and diagrams exported from
    presentation or diagramming software often do not.  Check the
    source file and add a profile before including it (\emph{e.g.}\ via
    GIMP's \emph{Image → Mode → Assign Colour Profile}, or with
    ImageMagick: \verb!magick -profile sRGB.icc input.jpg output.jpg!).

  \item[PNG] Inkscape embeds an sRGB profile when exporting via
    \emph{File → Export PNG Image}.  Images from other sources should be
    checked; if the profile is missing, add it with the same tools as
    for JPEG above.

  \item[TIFF] \LaTeX{} engines cannot include TIFF directly.  Convert
    to PDF or PNG with an embedded ICC profile using an image-editing
    tool (GIMP, ImageMagick, or Photoshop) before including the result.

  \item[PDF (figures)] An included PDF figure must itself use
    device-independent colour spaces throughout.  Figures produced by
    Matplotlib, Inkscape, and TikZ are almost always clean.  Scanned
    or converted PDFs may not be — if VeraPDF reports colour-space
    errors linked to a specific figure, pre-process it with Ghostscript
    before re-including:
\begin{bashcode}
gs -dBATCH -dNOPAUSE -sDEVICE=pdfwrite \
   -sColorConversionStrategy=sRGB \
   -o figure-fixed.pdf figure-original.pdf
\end{bashcode}
\end{description}

\noindent The safest choice is \textbf{vector PDF} (TikZ, Inkscape,
Matplotlib): it contains no raster data and is intrinsically
colour-space clean.

\medskip
\noindent\textbf{Opting out.}
To disable PDF/A output even in \texttt{final} mode, uncomment the escape
hatch in \verb!0-Config/1_novathesis.tex!:

\begin{latexcode}
\ntsetup{pdfa=false}
\end{latexcode}

\noindent\textbf{Verifying compliance.}
After the final build, validate with:
\begin{itemize}
  \item \textbf{VeraPDF} (open source, \url{https://verapdf.org}) —
        the reference PDF/A validator.
  \item \textbf{Adobe Acrobat Pro} — \emph{Tools → Print Production →
        Preflight → PDF/A compliance}.
  \item \textbf{PDF/A Live!} (online service) — quick check without local
        installation.
\end{itemize}
